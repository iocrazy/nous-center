# CEO Plan: nous-center v3 — Platform Upgrade

Generated by /plan-ceo-review on 2026-04-16
Branch: feature/nous-center-v2 | Mode: SCOPE EXPANSION
Repo: nous-center

## Vision

### Premise

现在的 nous-center 是"Ark 克隆 + vLLM 外壳" —— 一个优秀的 data plane。但上游 app（mediahub / 未来的 coze-alike / hermes-alike）要做记忆、成本观测、并行工作流时，发现 nous-center 没有暴露 ABC/契约，每个 app 都得重新发明。

v3 的目标：**把 nous-center 从 data plane 升级成 platform**，让上游 app 通过 ABC 而不是 HTTP 约定集成。

### 10x Check

不只是"上游能接入"，而是成为 Python 世界 AI infra 标杆：
- ABC 自带 reference 实现，零依赖可 prototype
- `nous-center-sdk` typed Python 客户端
- `/v1/models` 暴露 capabilities 让客户端智能路由
- `/plugins/memory/` 按 hermes 范式接受社区 PR

### Platonic Ideal

```python
# mediahub 工程师早上 9 点
pip install nous-center-sdk
from nous_center import Client

client = Client("http://localhost:8000", api_key="sk-...")
client.memory.attach(provider="pg")          # reference impl
resp = client.responses.create(
    model="qwen3.5",
    input="hi",
    previous_response_id=...,
)
print(resp.usage.cost)  # Decimal("0.00245")

if client.models.get("qwen3.5").capabilities.tool_use:
    resp = client.responses.create(..., tools=[...])
```

10 分钟从零到能跑。

## 实施计划（3 波发布）

### Wave 1 · Platform Contracts（~7.5 天，Eng Review 升级后）

**一次性立好所有 ABC，上游 app 接入不用等后续波。**

| # | 任务 | 来源 | 工作量 |
|---|------|------|--------|
| 1 | 事件扩展：增加 `node_end_streaming` / `workflow_interrupt` / `workflow_resume` / `function_call` / `tool_response` 等事件类型 | coze + hermes | 0.5d |
| 2 | MemoryProvider ABC：`initialize/shutdown/system_prompt_block/prefetch/sync_turn/on_session_end/on_pre_compress` + 生命周期 hooks | hermes | 1d |
| 3 | ContextEngine ABC：替换硬编码 `compact_messages`，抽象成可插拔压缩策略 | hermes | 1d |
| 4 | 节点接口分层：定义 `Invokable / Streamable / Collectable / Transformable` Protocol | coze | 1d |
| E1 | **PGMemoryProvider reference**：`memory_entries` + `memory_embeddings`（pgvector hook）+ 完整 ABC 实现 | EXPANSION | 1d |

**Ship milestone：** Wave 1 完成后，mediahub 可以立即开工实现三层记忆架构。

**Spec 产物：** `docs/superpowers/specs/2026-04-16-wave1-platform-contracts-design.md`

### Wave 2 · Observability & Reliability（~3 天）

**平台可靠性一次到位。**

| # | 任务 | 来源 | 工作量 |
|---|------|------|--------|
| 5 | ratelimit-remaining headers：response header 带 `x-ratelimit-remaining-{requests,tokens}` 12 个字段，SDK 客户端能自动退避 | hermes | 0.5d |
| 6 | CanonicalUsage + pricing YAML：`backend/configs/pricing/{model}.yaml` 描述单价；`/usage/inference` 升级返回 `cost` / `cost_status` / `cost_source` | hermes | 0.5d |
| 7 | Jittered backoff：`src/utils/retry.py` 通用重试层，workflow_executor 和 openai_compat 遇到 5xx 用它 | hermes | 0.5d |
| 8 | Checkpoint 表：`workflow_run_checkpoints(run_id, node_id, outputs_json, created_at)`，每个 `node_complete` 写一行；后端重启不丢 in-flight + 前端能历史重放 | coze | 1d |
| E3 | **/v1/models capabilities**：每个模型返回 `{tool_use, vision, thinking, streaming, max_len}`；从 config.json + models.yaml 综合推导 | EXPANSION | 0.5d |
